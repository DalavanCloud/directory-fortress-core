    • Setup Apache Tomcat Web Server



    This document contains instructions to install Tomcat 7. Any Tomcat (version 7 and beyond) instance may be used but server commands and file locations must be translated to match target environment.

    This section describes how to install Apache Tomcat to Debian and Fedora distros

    1. If Tomcat is already installed skip this step.

      Debian: 
                  
      
# sudo apt-get install tomcat7 tomcat7-admin tomcat7-docs
      Fedora (as root): 
                  
      
# mkdir tmp
# cd tmp
# wget http://www.us.apache.org/dist/tomcat/tomcat-7/v7.0.55/bin/apache-tomcat-7.0.55.tar.gz
# tar xzf apache-tomcat-7.0.55.tar.gz
# mv apache-tomcat-7.0.55 /usr/share/tomcat7
# cd /usr/share/tomcat7
# ./bin/startup.sh
    2. Verify process is running: 
                  
      
# ps -ef | grep tomcat
    3. Place the fortress-realm proxy jar from Build Apache Fortress Realm into Tomcat server's /lib folder. The proxy jar name is fortress-realm-proxy-[version].jar and located here: directory-fortress-realm/proxy/target 
                
      
# cp fortress-realm-proxy-[VERSION]-proxy.jar /usr/share/tomcat7/lib
      Where [VERSION] matches the value in this packages pom.xml file. At the time of this writing the current version was 1.0-RC40.

    4. Edit tomcat-users.xml: 
                
      
# vi /usr/share/tomcat7/conf/tomcat-users.xml
      Note: Depending on how and where you installed Tomcat, the tomcat-users.xml could be different locations. To find out where: 
                
      
# sudo find / -name tomcat-users.xml
    5. Add privilege for tcmanager user to connect with Tomcat Manager. This account can access GUI and deploy via maven. 
                  
      
<role rolename="manager-script"/>
<role rolename="manager-gui"/>
<user username="tcmanager" password="m@nager123" roles="manager-script,manager-gui"/>
      Using the config file realm to store credentials for an important application like the Tomcat Manager is a security antipattern. For production, a realm like fortress realm should used globally by Tomcat.

    6. Save and exit.

    7. Restart Tomcat on Debian: 
                  
      
# sudo /etc/init.d/tomcat7 restart
      Restart Tomcat on Fedora: 
                  
      
# ./bin/shutdown.sh
# ./bin/startup.sh
    8. Verify clean logs after startup: 
                  
      
# tail -f -n10000 /var/lib/tomcat7/logs/catalina.out
      Ensure there are no errors within Tomcat's log. One common problem: UnsupportedClassVersionError. This is a sure sign that your Tomcat is running under the wrong version of java. Remember Java 7 or greater is required to run this tutorial.
    9. Verify setup by signing onto the Tomcat Manager app with credentials userId: tcmanager, password: m@nager123

      http://localhost:8080/manager
