This package uses Apache Commons Configuration APIs to provide configuration data to fortress along with CRUD APIs for storing config data on the ldap server.

The us.jts.fortress.config can store its data both in a flat file or within the LDAP directory itself. The oamConfig.xml contains the location of the bootstrap property file. The bootstrap properties can then point to a remote LDAP configuration node, called a 'realm', and referenced with this property: config.realm. An empty or null value for the config.realm property means the properties will all be stored locally in the file specified within oamConfig.xml. The package also contains entities and apis that are used to perform CRUD on remote configuration parameters. See the corresponding javadoc contained with this package for more info.
